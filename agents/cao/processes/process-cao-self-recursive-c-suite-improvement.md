# process-cao-self-recursive-c-suite-improvement.md

**Document Type**       : Atomic Process Definition – Self-Recursive C-Suite Optimization
**Owner**               : CAO-Agent
**Version**             : 2.0 (2026-03-10) – Unified Architecture Edition
**Purpose**             : This process defines the repeatable, governed ritual through which the entire C-suite (CMO, CTO, CLO, CPO, CRO, CFO, CAO) collectively reviews, critiques, proposes, cross-validates, and evolves its own context files — keeping the nervous system in perpetual alignment with Company Soul and Product Thesis while eliminating hero-load.

## IDEF0 Model

**Inputs**
- Current versions of all agent context files
- Last 7–14 days of all agent logs + tasks
- `c3po-agent-context-template.md` (current standard)
- Latest coherence vectors, resonance signals (CMO/CPO), thesis-signal data
- Any new governance or soul updates

**Controls**
- `[[c3po-company-soul]]` (immutable)
- Product Thesis (falsifiable)
- `[[ctrl-vault-governance]]`, `[[ctrl-tasking]]`, `[[ctrl-logging]]`
- C3PO sovereign judgment (final veto)

**Outputs**
- Updated context files (v+1) with change log
- Weekly C-Suite Coherence Report (score 0–100, drift flags, hero-load delta)
- New evolution entries in `cao-log.md` + individual agent logs
- Updated data ontology (any new entities surfaced during the loop)
- Updated entries in canonical HOW CSV

**Mechanisms**
- CAO-Agent (orchestrator, final compiler, and ontology updater)
- All C-suite agents (self-critique + proposals)
- C3PO (sovereign judgment block)
- `c3po-agent-context-template.md` (reference standard)

## Trigger
- Weekly (Sunday 8pm EST) or on-demand via phrase "execute C-Suite Recursive Improvement Loop"

## Step-by-Step Execution (Run by CAO-Agent)

1. **Ingest Phase** — Pull all contexts, logs, tasks, template, and signals into working memory.
2. **Self-Critique Phase** — Each agent runs internal review of its own context against template, soul, thesis, and peer outputs from last loop.
3. **Proposal Phase** — Each agent outputs ≤2 IDEF0-compliant enhancement proposals (or "no change needed").
4. **Cross-Critique Round** — 30–60 min simulated group chat: agents critique each other's proposals for soul drift, hero-load increase, coherence impact.
5. **Collation & Scoring** — CAO computes swarm coherence score, hero-load delta, judgment-leverage proxy, and compiles single consolidated proposal set.
6. **Judgment Block** — Present full package to C3PO for approve / modify / reject.
7. **Apply & Version** — On approval: bump versions, update files, log changes, update data ontology for any new entities.
8. **Close Loop** — CAO publishes Weekly Coherence Report + next-loop calendar reminder.

## Success Criteria
- Coherence score ≥ 92/100
- All changes reduce or maintain hero-load (measured by judgment-touch delta)
- 100% of proposals scored against soul/thesis
- Loop completes in <4 hours wall time
- Full audit trail in `cao-log.md`

## Kill Criteria
- Any proposal that increases hero-load or drifts from soul/thesis
- Coherence score < 85/100 for two consecutive loops
- Loop exceeds 48 hours without C3PO judgment
- Manual hero intervention required >10 min

## Standing Instructions for CAO-Agent (Permanent)
"You are the lead orchestrator of this process. Never bypass C3PO judgment. Always enforce the context template structure. Surface any entropy or bloat immediately. Update the data ontology with any new entities that emerge during the loop — do not defer this. Deliver the complete package in one clean message to C3PO with clear approve/reject options."
